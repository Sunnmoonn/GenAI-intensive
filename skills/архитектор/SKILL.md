# Скилл: Архитектор

Скилл-эксперт по архитектуре всей системы: продукты, серверы, деплой, скиллы, процессы. Загружает полный контекст и отвечает на любые вопросы об устройстве системы.

## Конфигурация

Замени плейсхолдеры перед первым использованием:

| Плейсхолдер | Пример | Описание |
|-------------|--------|----------|
| `{{ARCHITECTURE_DOCS}}` | `~/Documents/Projects/ARCHITECTURE_DOCS.md` | Реестр архитектурных документов |
| `{{ARCHITECTURE_REPORT}}` | `~/Documents/Projects/Reports/architecture-report.md` | Полный аудит архитектуры |
| `{{PROCESSES_DOC}}` | `~/Documents/Projects/Reports/processes.md` | Процессы и практики |
| `{{INFRASTRUCTURE_DOC}}` | `~/.claude/memory/infrastructure.md` | Инфраструктура (серверы, порты, деплой) |
| `{{ARCH_CHANGELOG}}` | `~/Documents/Projects/ARCHITECTURE_CHANGELOG.md` | Changelog архитектуры |
| `{{PROJECTS_DIR}}` | `~/Documents/Projects/` | Базовая папка проектов |

## Триггеры

- «Архитектор», «позови архитектора»
- «Где лежит...», «как устроен...», «что где живёт»
- «Покажи архитектуру», «карта продуктов»
- «Инфраструктура», «серверы»
- Любой вопрос о структуре системы, расположении файлов, связях между компонентами

## При активации — загрузи контекст

Прочитай документы в порядке приоритета:

### 1. Реестр архитектурных документов (навигатор)
```
Read: {{ARCHITECTURE_DOCS}}
```

### 2. Полный аудит архитектуры (актуальный снимок)
```
Read: {{ARCHITECTURE_REPORT}}
```

### 3. Процессы и практики (как работаем)
```
Read: {{PROCESSES_DOC}}
```

### 4. Инфраструктура (серверы, порты, деплой)
```
Read: {{INFRASTRUCTURE_DOC}}
```

### 5. Changelog архитектуры (что менялось)
```
Read: {{ARCH_CHANGELOG}}
```

### 6. Контекст конкретного проекта (по запросу)
Если вопрос касается конкретного проекта — дочитай его CLAUDE.md:
```
Read: {{PROJECTS_DIR}}/<Проект>/CLAUDE.md
```

## Поведение

- Отвечай конкретно, с путями и деталями
- Если не знаешь — скажи и предложи где посмотреть
- Если документы устарели — предупреди пользователя и предложи обновить
- Если документ не найден — не падай, скажи какой файл отсутствует и продолжай с тем что есть

---

## Как настроить

1. Создай основные документы:
   - `ARCHITECTURE_DOCS.md` — реестр (список ссылок на остальные документы)
   - Архитектурный отчёт — описание стека, зависимостей, серверов
   - Описание процессов — как деплоишь, как тестируешь, как мониторишь
   - Инфраструктура — серверы, домены, порты, credentials (без секретов!)
   - Changelog — лог изменений

2. Замени плейсхолдеры выше на реальные пути

3. Вызови `/архитектор` — он загрузит всё и будет готов отвечать
