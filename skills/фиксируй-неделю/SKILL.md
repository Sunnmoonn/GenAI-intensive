# Скилл: ФИКСИРУЙ-НЕДЕЛЮ

## Конфигурация

Замени плейсхолдеры перед первым использованием:

| Плейсхолдер | Пример | Описание |
|-------------|--------|----------|
| `{{SESSION_LOG}}` | `~/Documents/Projects/SESSION_LOG.md` | Путь к логу сессий |
| `{{WEEKLY_LOG}}` | `~/Documents/Projects/WEEKLY_LOG.md` | Путь к недельным итогам |

---

## Файлы

**Источник:** `{{SESSION_LOG}}`
**Итоги:** `{{WEEKLY_LOG}}`

---

## Алгоритм

### Шаг 1: Найти дату последнего итога

Прочитай WEEKLY_LOG (первые 50 строк):
```
Read: {{WEEKLY_LOG}}
limit: 50
```

- Ищи строки вида `## YYYY-MM-DD` или `**Период:**` — они содержат дату последнего итога
- Если файл не существует или пустой -> дата последнего итога = **самая ранняя дата в SESSION_LOG**
- Запомни эту дату как `FROM_DATE`

### Шаг 2: Вытащить все заголовки сессий с датой

```bash
grep -n "^## 20" "{{SESSION_LOG}}"
```

Отфильтруй только те, у которых дата >= FROM_DATE.

### Шаг 3: Прочитать содержимое нужных сессий

По номерам строк — прочитай блоки от каждого заголовка до следующего `---`.

Если сессий много (>10) — читай порциями по 300-400 строк:
```
Read: SESSION_LOG, offset=<строка>, limit=400
```

### Шаг 4: Построить итог

Проанализируй прочитанные сессии и составь итог.

**Что считать:**
- Количество сессий за период
- Список проектов (уникальные)
- Созданные скиллы
- Деплои
- Созданные артефакты (файлы, лендинги, документы)

**Главный вопрос для каждого проекта:** *Что конкретно продвинулось? Какой артефакт появился?*

### Шаг 5: Записать в WEEKLY_LOG

Если файл не существует — создай его с заголовком:
```markdown
# Недельные итоги

---
```

Добавь новый блок **В НАЧАЛО** (после заголовка, перед предыдущими итогами).

---

## Формат итога

```markdown
## YYYY-MM-DD — YYYY-MM-DD | Итог за период

**Период:** <FROM_DATE> -> <TODAY>
**Сессий:** <число>
**Проекты:** <через запятую>

---

### По проектам

#### <Название проекта>
- **Главное:** <1-2 предложения — что сдвинулось>
- Ключевые артефакты: <список>
- Статус: <в работе / запущен / пауза>

#### <Следующий проект>
...

---

### Инфраструктура и скиллы

- Новые скиллы: `/скилл-1`, `/скилл-2`
- Инфраструктура: <VPS, домены, инструменты>
- Методология: <если были методологические сессии>

---

### Рефлексия

**Темп:** <высокий / средний / низкий>
**Фокус:** <был ли фокус или распыление?>
**Главное достижение периода:** <одна фраза>
**Что не двигалось:** <проекты в стагнации>
**Вопросы на следующий период:** <что стоит решить или проверить>

---
```

---

## Правила

1. **WEEKLY_LOG** — отдельный файл, SESSION_LOG не трогать
2. Новые итоги — **сверху**, старые остаются ниже
3. Рефлексию писать честно — не хвалить и не критиковать без оснований, просто констатировать факты
4. Если период > 14 дней — разбить на подпериоды по неделям внутри блока
5. После записи — сообщить пользователю: период, кол-во сессий, файл сохранён
