# PDF / PPTX / DOCX → Markdown

Конвертирует документы в чистый Markdown через [MarkItDown](https://github.com/microsoft/markitdown) от Microsoft.

**Поддерживаемые форматы:** PDF, PPTX, DOCX, XLSX, XLS, HTML, изображения (JPG, PNG), ZIP-архивы.

## Триггеры

- «pdf в markdown», «pptx в markdown», «конвертируй PDF/PPTX/DOCX»
- «переведи PDF в текст», «из PDF в текст»
- «извлеки текст из файла», «конвертируй документ»
- `/pdf-to-md`

## Алгоритм

### Шаг 1: Определи входные файлы

Если пользователь передал путь — используй его.
Если путь не передан — спроси:

```
Укажи путь к файлу (PDF, PPTX, DOCX, XLSX, изображение).
Можно перетащить файл прямо в чат или вставить полный путь.
```

### Шаг 2: Проверь и установи markitdown

```bash
if ! command -v markitdown &>/dev/null; then
  echo "Устанавливаю markitdown..."
  pip install 'markitdown[all]' -q
fi
markitdown --version 2>/dev/null || echo "ok"
```

Если pip недоступен — попробуй `pip3` или `python3 -m pip`.

### Шаг 3: Конвертируй файл

```bash
INPUT="/path/to/file.pdf"
OUTPUT="${INPUT%.*}.md"
markitdown "$INPUT" > "$OUTPUT"
echo "Готово: $OUTPUT"
echo "Размер: $(wc -l < "$OUTPUT") строк"
```

**Если нужна папка назначения** (пользователь указал куда сохранить):
```bash
OUTPUT="/target/dir/$(basename "${INPUT%.*}").md"
markitdown "$INPUT" > "$OUTPUT"
```

### Шаг 4: Покажи превью

Показывай первые 60 строк результата через Read tool.

Если файл пустой или < 10 строк — сообщи:
> PDF, вероятно, состоит из сканов без OCR. MarkItDown не поддерживает OCR.
> Варианты: 1) Использовать Marker (тяжелее, но OCR) 2) Использовать Adobe или другой OCR

```
КОНВЕРТАЦИЯ ЗАВЕРШЕНА

```

## Пакетная конвертация

Если пользователь передал несколько файлов или папку:

```bash
# Несколько файлов
for f in /path/to/*.pdf; do
  markitdown "$f" > "${f%.*}.md"
  echo "✓ $(basename $f)"
done
```

## Важные правила

- **OCR:** MarkItDown НЕ делает OCR. Сканы без текстового слоя дадут пустой файл — предупреди пользователя.
- **PPTX:** Извлекается текст слайдов, заголовки, таблицы. Изображения из слайдов описываются через LLM если передан `--llm-client` (по умолчанию не используется).
- **Перезапись:** Если `<name>.md` уже существует — предупреди перед перезаписью.
- **Пути:** Использовать абсолютные пути. Если пользователь дал относительный — уточнить рабочую директорию.
