# ЦА-Прогон — Симуляция аудитории через мультиагентов

Универсальный мультиагентный прогон любого артефакта через симулированную ЦА.

## Конфигурация

Замени плейсхолдеры перед первым использованием:

| Плейсхолдер | Пример | Описание |
|-------------|--------|----------|
| `{{OUTPUT_DIR}}` | `~/Documents/Projects/audience-tests` | Папка для сохранения результатов прогонов |

---

## Когда вызывать

- `/ца-прогон`
- «обкатай на ЦА», «прогони через аватары», «что подумает аудитория»
- После создания: лендинга, поста, фичи, КП, продуктовой карточки, демо
- Когда нужна не одна оценка, а 5-10 разных перспектив

---

## АЛГОРИТМ

### Фаза 1: ЗАХВАТ ЗАДАЧИ

Прочитай контекст. Если артефакт уже в разговоре (файл, ссылка, текст) — используй его. Иначе спроси через AskUserQuestion одним блоком:

1. **Что прогоняем?** (путь к файлу / URL / текст)
2. **Какая аудитория?** (если не очевидно — например «продакт-менеджеры в IT», «родители 30-45», «B2B-продажники»)
3. **Сколько аватаров?** (по умолчанию 5; диапазон 3-10)
4. **На что фокус?** (общая реакция / конверсия / цена / методология / эмоция — что важно)

### Фаза 2: ПРОЕКТИРОВАНИЕ АВАТАРОВ

На основе ЦА сгенерируй N аватаров **с РАЗНООБРАЗИЕМ** по осям:
- Возраст / пол
- Грейд / опыт
- Сегмент (новичок / эксперт / скептик / евангелист)
- Контекст (геолокация, размер компании, бюджет, боль)
- Эмоциональная позиция (восторженный / усталый / циничный / любопытный)

Для каждого аватара заполни карточку:
```
Аватар N: Имя, возраст, роль (город, доход)
- Кто: 1 предложение
- Боль: что болит сейчас
- Контекст: почему здесь
- Стоп-слова: что отталкивает
- Что хочет увидеть: ожидание
```

**Покажи аватары пользователю** одним блоком и спроси «Запускаем прогон?» через AskUserQuestion.

### Фаза 3: ПОДГОТОВКА АРТЕФАКТА

Если артефакт — файл, прочитай его (Read). Если URL — WebFetch. Подготовь компактное описание (до 2000 слов), которое будет передаваться каждому агенту.

Создай рабочую папку:
```
{{OUTPUT_DIR}}/YYYY-MM-DD_slug/
  ├── 00_artifact.md       # копия артефакта
  ├── 01_avatars.md        # карточки аватаров
  ├── 02_brief.md          # бриф для агентов
  ├── responses/           # ответы аватаров
  │   ├── avatar_1.md
  │   └── ...
  └── 03_synthesis.md      # синтез (Фаза 5)
```

### Фаза 4: ПАРАЛЛЕЛЬНЫЙ ПРОГОН (мульти-Agent)

**Запусти все аватары ОДНИМ сообщением с N параллельных Agent-вызовов** (subagent_type: general-purpose).

Промпт каждому агенту:
```
Ты — [полная карточка аватара]. Не выходи из роли.

Тебе показали следующий артефакт:
---
[содержимое артефакта]
---

Задача: дай развёрнутую честную реакцию от первого лица.

Структура ответа:
1. **Первая реакция** (3 секунды — что бросается в глаза)
2. **Что нравится** (конкретно)
3. **Что раздражает / непонятно / не верю** (конкретные цитаты)
4. **Что хочу увидеть, но нет**
5. **Действие**: пошерю / закрою / куплю / проигнорирую — и почему
6. **Оценка по 10-балльной**: релевантность / доверие / желание поделиться

Будь резким и честным. Не льсти. Если плохо — говори «плохо». Используй язык своей роли (сленг, профессионализмы).

Формат: markdown, ~400-600 слов.
```

После выполнения каждый агент возвращает фидбек — сохрани в `responses/avatar_N.md`.

### Фаза 5: СБОР И АНАЛИЗ

Собери все ответы. Создай файл `02_raw_feedback.md`:
- Все N ответов подряд
- Без интерпретации, без выводов

Создай файл `03_patterns.md` — **только наблюдения, не рекомендации**:
- Таблица «Проблема x кто отметил x критичность»
- Цитаты-якоря (3-5 самых ярких)
- Что хвалят единодушно
- Что критикуют единодушно
- Где аватары расходятся

Покажи пользователю саммари:
- Прогнано N аватаров
- Папка: ...
- Топ-3 проблемы (одна строка каждая)
- Топ-3 что хвалят

### Фаза 6: ОСТАНОВКА — СМЕНА МОДЕЛИ

**КРИТИЧНО: остановись здесь.** Скажи пользователю:

```
Сырая фактура собрана.

Прежде чем писать рекомендации — переключись на более умную модель
(Opus через /model или Fast mode для глубокого синтеза).

Когда переключишься, скажи «продолжай» или «пиши рекомендации» —
и я перейду к Фазе 7.
```

**НЕ генерируй рекомендации в этом сообщении.** Жди явного «продолжай».

### Фаза 7: РЕКОМЕНДАЦИИ (после смены модели)

Когда пользователь говорит продолжать:
1. Перечитай `02_raw_feedback.md` и `03_patterns.md`
2. Напиши `04_recommendations.md` со структурой:
   - **Что чинить срочно** (единодушные провалы) — 3-5 пунктов
   - **Что улучшить** (частые жалобы) — 3-5 пунктов
   - **Что усилить** (что хвалят, можно сделать ещё ярче) — 2-3 пункта
   - **Стратегические развилки** (если фидбек обнажил проблему позиционирования)
   - **Конкретные правки** — список изменений в артефакте с приоритетом
3. Покажи пользователю с путём к файлу

## ПРИНЦИПЫ

- НЕ генерируй аватары шаблонно («Маша 25, маркетолог» 5 раз). Каждый аватар — отдельный человек с уникальной позицией.
- НЕ объединяй ответы агентов в один мега-аватар. Сохраняй индивидуальность.
- НЕ пиши рекомендации до смены модели — это main-агент должен делать на более мощной модели.
- Параллельные Agent-вызовы в одном сообщении (не последовательно).
- Сохраняй ВСЁ на диск — фактура должна пережить компактизацию контекста.
- Аватары могут противоречить друг другу — это норма, не сглаживай.
