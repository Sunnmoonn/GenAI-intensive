## Ключевые решения, принятые за эти две недели

- **Multi-tenant через Supabase RLS + JWT custom claims** — каждая организация видит только свои данные; функция `auth_org_id()` извлекает `org_id` из JWT, все политики строятся на ней. Альтернатива (фильтр в коде) отклонена — утечка данных при ошибке разработчика слишком высокий риск для B2B SaaS.
- **Soft delete вместо физического удаления** — все сущности (projects, tasks, reagents, consumables) помечаются `deleted_at`, а не стираются. Причина: audit trail обязателен для GxP-клиентов.
- **Единая миграция 001 вместо множества мелких** — на этапе MVP проще держать всю схему в одном файле; разбивка на шаги добавлена в план.

## Сделано за две недели

- Полная схема БД: organizations → profiles → projects → project_members → tasks → reagents → consumables → audit_log
- Edge Function `custom-access-token` — добавляет `org_id` в JWT при логине
- DB trigger `on_auth_user_created` — автоматически создаёт профиль при регистрации
- RLS политики на всех таблицах (select / insert / update)
- Подключение фронтенда к реальным данным: Dashboard, Projects, Gantt, Reagents, Consumables, Profile
- Мобильная адаптация: bottom nav, адаптивные гриды, скроллируемые фильтры
- Деплой на Vercel + SPA-роутинг (rewrite rules в `vercel.json`)

## Что пришлось переделать

- **Gantt (6 итераций)** — синхронизация высот строк с динамическим контентом оказалась нетривиальной; переходили от фиксированной высоты к `h-16`, потом к обёртке имён задач. Бейдж статуса трижды менял позицию (текст → иконка справа вверху → мобильная левая колонка). До: строки съезжали при wrap. После: стабильная высота `h-16` с `flex-wrap`.
- **setCurrentUserId** — исправлена сигнатура updater-функции в store после TypeScript-ошибки при подключении реального профиля пользователя.
