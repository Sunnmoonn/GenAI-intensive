# PERSONAL — Личная AI-продуктивность

## Мои скилы

Формат: `[имя скилла] — когда триггерится — что делает — какой контекст подкладывает`

- `benchling-arch-research` — когда нужно разложить конкурента по слоям и найти moat — делает структурированный анализ фронта / бэка / данных / инфры по публичным источникам — подкладывает название сервиса, домен, что именно хочу воспроизвести
- `ui-enterprise-sci-saas` — когда нужно сгенерировать или отревьюить UI для Квантары — применяет дизайн-систему (цвета, типографика, компоненты, anti-patterns) в духе Enterprise Scientific SaaS — подкладывает полный UI blueprint
- `venture-scan-rf` — когда нужно оценить зарубежный стартап на применимость к рынку РФ — считает скор по формуле M²×T×A×B×P и выдаёт цветную маркировку — подкладывает методологию с описанием факторов и примерами расчётов
- `firecrawl-market-research` — когда нужно собрать данные о рынке или конкурентах из открытых источников — скрейпит сайты, ищет отчёты, извлекает структурированные данные — подкладывает URL или поисковый запрос + нужный формат вывода
- `project-clone-workflow` — оглавление набора из 6 скиллов для клонирования сервиса от поиска до публикации — см. отдельные файлы ниже
- `clone-1-search` — когда нужно выбрать сервис под клонирование из длинного списка кандидатов — скорит по VENTURE-SCAN RF, сравнивает 20+ сервисов, выдаёт финальный выбор с обоснованием — подкладывает нишу, ограничения стека, горизонт MVP
- `clone-2-analysis` — когда сервис выбран и нужно разложить его на слои — Firecrawl-скрейп + arch-research промпт → стек, UI-паттерны, moat — подкладывает URL, домен, что именно воспроизводить
- `clone-3-spec` — когда анализ готов и нужно написать ТЗ — структурирует модули, стек, ров в PROJECTCLONE/README.md — подкладывает результат анализа + временной горизонт
- `clone-4-implementation` — когда спек готов и нужно собрать MVP — порядок: БД+Auth → страницы → мобильная адаптация — подкладывает спек, Supabase credentials, дизайн-систему
- `clone-5-testing` — когда MVP собран и нужно проверить перед публикацией — чек-лист: RLS-изоляция, мобильный вид, edge cases — подкладывает Preview URL и список модулей
- `clone-6-publish` — когда тестирование пройдено и нужно выложить в прод — GitHub без секретов + Vercel SPA-конфиг + Supabase prod — подкладывает Vercel account, env vars

(Сам код скилов — в [skills/](skills/).)

## Инструменты

- [`startup-market/`](startup-market/) — Next.js-приложение для скоринга зарубежных стартапов: скрейпит YC/ProductHunt/Crunchbase/Telegram, скорит по VENTURE-SCAN RF 2.0, генерирует еженедельный AI-дайджест через Claude API. Используется для поиска идей под клонирование и мониторинга трендов.

## Задачи, которые автоматизирую

- Анализ нового конкурента — раньше часами читала блоги и доки, теперь один промпт → структурированный readout
- Генерация UI под Квантару — без скилла каждый раз объясняла дизайн-систему с нуля
- Оценка стартапов для анализа рынка — методология VENTURE-SCAN RF даёт сопоставимые скоры вместо субъективных суждений
