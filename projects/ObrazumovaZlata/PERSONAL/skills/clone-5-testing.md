# Скилл: Clone Step 5 — Тестирование

## Триггер

Когда MVP собран и нужно убедиться, что он работает перед публикацией.

## Что делает

Прогоняет MVP через чек-лист сценариев с фокусом на безопасность данных (RLS) и мобильный вид.

## Чек-лист

### Критичные (блокируют публикацию)
- [ ] Регистрация → автосоздание профиля → вход в аккаунт
- [ ] RLS-изоляция: создать двух пользователей в разных org → убедиться, что данные не пересекаются
- [ ] CRUD по всем сущностям: создать / редактировать / удалить запись
- [ ] Выход из аккаунта → редирект на /login

### Важные
- [ ] Мобильный вид (Chrome DevTools 375px): навигация, таблицы, формы
- [ ] Пустые состояния: новый аккаунт без данных — нет ошибок, есть CTA
- [ ] Ошибки сети: отключить интернет → приложение не падает, показывает сообщение

### Желательные
- [ ] Просроченная сессия → автоматический редирект на логин
- [ ] Реальное мобильное устройство (не только DevTools)

## Инструменты

- Vercel Preview URL — каждый коммит получает preview автоматически
- Chrome DevTools → вкладка Network (проверить RLS запросы) + Device toolbar
- Supabase Dashboard → Table Editor (проверить данные напрямую)

## Контекст для подкладки

- Preview URL от Vercel
- Список модулей из спека (чтобы не пропустить ни один)
