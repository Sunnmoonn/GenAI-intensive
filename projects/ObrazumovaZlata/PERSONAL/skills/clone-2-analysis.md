# Скилл: Clone Step 2 — Анализ интерфейса и архитектуры

## Триггер

Когда сервис для клонирования выбран и нужно разложить его на слои перед написанием спека.

## Что делает

Извлекает технический стек, UI-паттерны и конкурентный moat из публичных источников без доступа к исходному коду.

## Шаги

1. **Firecrawl-скрейп:** главная, pricing, docs, changelog, blog → конкурентная матрица (фичи / цена / позиционирование)
2. **Arch-research промпт** (`benchling-arch-research`): engineering blog, job postings, API docs → извлечь frontend/backend/data/infra стек
3. **UI-анализ:** пройти по всем публичным скриншотам и демо-видео, зафиксировать:
   - Компоненты верхнего уровня (навигация, layout, дашборды)
   - Паттерны взаимодействия (inline edit, bulk actions, фильтры)
   - Мобильная адаптация: есть или нет
4. **Moat-анализ:** где живёт настоящее конкурентное преимущество — данные / switching cost / сеть / регуляторка
5. **Список экранов MVP:** что воспроизводить в первую очередь, что срезать

## Контекст для подкладки

- URL сервиса
- Домен (ELN/LIMS, fintech, HR — влияет на специфику анализа)
- Что именно хочешь воспроизвести: визуал, данные, архитектуру

## Пример

Benchling → выявлено: React+Redux фронт, Postgres источник истины, REST API, три поколения search-архитектуры. Moat — не технология, а накопленные данные и switching cost.
