# Скилл: Clone Step 4 — Реализация ТЗ

## Триггер

Когда спек готов и нужно собрать рабочий MVP.

## Что делает

Переводит спек в рабочий код через связку Claude Code + Cursor + Supabase в фиксированном порядке.

## Порядок сборки

### 1. Схема БД и Auth (день 1)
- Спроектировать multi-tenant схему: organizations → profiles → [entities]
- Написать SQL миграцию с RLS-политиками через Claude Code
- Настроить Supabase Auth + Edge Function для JWT custom claims (org_id в токене)
- Проверить: пользователи разных org не видят данные друг друга

### 2. Страницы в порядке приоритета (дни 2–8)
Для каждого модуля из спека:
1. Описать экран словами (layout, компоненты, поведение)
2. Сгенерировать через Claude Code → React/TypeScript компонент
3. Подключить к Supabase (типы из `supabase gen types`)
4. Cursor: рефакторинг, выравнивание под проектные типы

### 3. Мобильная адаптация (день 9–10)
- Bottom navigation вместо sidebar
- Адаптивные гриды (`grid-cols-1 sm:grid-cols-2`)
- Горизонтальный скролл для таблиц и Gantt

## Инструменты

| Задача | Инструмент |
|---|---|
| Компоненты по описанию экрана | Claude Code |
| Рефакторинг и типы | Cursor |
| БД, RLS, Auth, Edge Functions | Supabase + Claude Code |
| Итеративные правки UI | Claude Code (в диалоге) |

## Контекст для подкладки

- Спек из `clone-3-spec`
- Supabase project URL + anon key (для подключения)
- Дизайн-система (если есть — скилл `ui-enterprise-sci-saas`)

## Пример

Квантара: схема БД за день 1 → 7 страниц за неделю → Gantt потребовал 6 итераций (синхронизация высот строк). Самое сложное — динамические высоты при flex-wrap.
