# PROJECTCLONE — Клонирование doomlingo.ai

## 1. Выбранный сервис

**[Doomlingo](https://www.doomlingo.ai/)** — приложение, которое превращает привычку бездумного скроллинга ленты в изучение языка. Короткие видео + субтитры на двух языках + свайп вверх/вниз.

**Почему выбрал:** я сам залипаю в TikTok по вечерам. Если бы в этот момент в голову попадала хоть одна новая фраза на английском — время перестало бы быть «потерянным». Doomlingo нашёл хак: не бороться с привычкой, а подменить её содержимое.

**Моя гипотеза:** для русскоязычного пользователя нужен не учебный, а «живой» контент — реальные интервью и разговоры, где английский встречается органически, а не как грамматическое упражнение.

**Клон:** [Вникай](vnikay/) — веб-версия (React + TypeScript + Vite). Слоган: «Учи английский, залипая в телефон».

## 2. Скилы, которыми собирал

- `frontend-design` — весь UI: iPhone-мокап, dark-theme с фиолетовым свечением, компоновка «телефон слева / субтитры справа», глассморфные контролы
- `feature-dev` — логика синхронизации субтитров по `currentTime` видео, fade-переходы, мульти-ввод (touch-свайп + wheel)
- Whisper (OpenAI, small) — транскрибация 10 собственных видео с тайминками, дальше ручной перевод RU→EN
- Claude Code как основной инструмент — я непрограммист, весь код писался в диалоге

## 5. Как собирал (с позиции непрограммиста)

1. **Контент первым.** Записал 10 коротких видео на телефон, потом через `ffmpeg` + Whisper-small получил транскрипт с таймкодами и сам перевёл на английский. Без контента UI было некуда упаковывать.
2. **Мокап вместо «страницы».** Первая версия выглядела как обычный сайт с большим видео — не работало, атмосфера TikTok терялась. Переделал в iPhone-фрейм 300×610 с notch — сразу стало «то самое».
3. **Субтитры сбоку, а не поверх.** Изначально субтитры шли поверх видео — мешали смотреть. Вынес их в правую колонку (260px) — читаются и не закрывают кадр.
4. **Два инпута навигации.** На мобилке — свайп, на десктопе — колесо. Wheel без cooldown переключал по 5 сцен за один скролл тачпада, добавил 800ms блокировку.
5. **Fade без мигания.** Наивный `setState` по каждому `timeupdate` мигал на переходах. Через `useRef` для предыдущего субтитра и 150ms opacity-анимацию — стало плавно.

## 6. Что было тяжело

- **Синхронизация субтитров.** `onTimeUpdate` стреляет ~4 раза в секунду — без мемоизации предыдущего состояния на каждом тике ререндерился весь блок субтитров, появлялось мерцание. Решение: `prevSubtitleRef` + сравнение по `start`.
- **Wheel vs touch.** На ноутбуке с тачпадом `deltaY` приходит маленькими порциями и инерцией — нужен и порог `|deltaY| > 30`, и cooldown, иначе один жест = 3 сцены.
- **Видео на GitHub.** 99MB MP4 — на грани лимита. Не стал уходить в LFS, оставил как есть, но в проде это однозначно CDN.

## 7. Что осталось открытым

- Автоматизация контент-пайплайна: Whisper + автоперевод + автонарезка — чтобы добавить сцену занимало минуты, а не час
- Уровни сложности: сейчас все 10 видео равнозначны, нет градации A2/B1/B2
- Сохранение прогресса и словаря пользователя — пока stateless

## 8. ⭐ Ров

**Ров 1 — Контент-библиотека.** Размеченное видео (таймкоды + двуязычные субтитры) — дорогой актив. Даже с Whisper качество ручной сверки остаётся барьером: 10 видео = ~2 часа работы. Новый игрок не соберёт 10k единиц за месяц.

**Ров 2 — Данные о понимании.** Каждый свайп, пауза, повтор — сигнал: «слишком сложно / слишком легко». Накопив такой поведенческий профиль, сервис подбирает следующее видео точнее любого внешнего теста. Этот профиль невозможно экспортировать.

**Ров 3 — Привычка и streak.** Классический moat из Duolingo: страх потерять streak и «выученные» слова. Чем дольше пользователь с нами, тем дороже уходить — не из-за сервиса, а из-за прогресса в голове.
