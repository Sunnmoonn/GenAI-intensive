# PA_Moat — рвы и защитные механизмы

## Когда запускать

Финальный шаг после `PA_Surface` и `PA_Mechanism`, либо автономно — если нужно быстро оценить защитные механизмы конкурента по короткому описанию.

## Что делает

Находит рвы продукта, приоритизирует их по силе и устойчивости на горизонте 12 месяцев. Отвечает на вопрос «почему новый игрок с такой же идеей не заберёт рынок за полгода».

## Типы рвов (чеклист)

Проверить каждый из 7 типов — есть / нет / слабый:

1. **Данные** — накопленный дата-сет, без которого продукт хуже работает.
2. **Сеть (network effect)** — ценность для пользователя растёт с количеством пользователей.
3. **Switching cost** — пользователь теряет прогресс/интеграцию при уходе.
4. **Привычка / бренд** — использование стало рутиной или именем нарицательным.
5. **Технология / железо** — ноу-хау, патент, сложный hardware-стек, который не копируется за квартал.
6. **Дистрибуция** — эксклюзивные каналы, партнёрства, каналы продаж, недоступные конкурентам.
7. **Регуляторика** — лицензии, сертификаты, барьер входа со стороны государства.

## Шаги

1. Для каждого типа рва — отметить статус (есть / слабый / нет) и 1–2 факта, которые это подтверждают.
2. Выбрать **2–3 главных рва**, на которых держится продукт реально, а не на словах.
3. Для каждого выбранного — сформулировать:
    - **Как накапливается** (механизм, который превращает время/деньги/пользователей в ров).
    - **Метрика-индикатор** — что показывает, что ров крепнет.
    - **Что его разрушит** (регуляция, технология, смена поведения).
4. **Сопоставление с конкурентами.** У кого какие рвы сильнее. Итог — на каком поле мы можем выиграть.

## Выход

```
# <Продукт> — Moat

## Чеклист (7 типов)
- Данные: ...
- Сеть: ...
- Switching cost: ...
- Привычка: ...
- Технология: ...
- Дистрибуция: ...
- Регуляторика: ...

## Ключевые рвы
### Ров 1 — <тип>
- Как накапливается: ...
- Метрика: ...
- Что разрушит: ...

## Сравнение с конкурентами
| Игрок | Данные | Сеть | ... |
| ... | ... | ... | ... |

## Вывод
- На каком поле играем: ...
- Что усиливать в следующие 3–6 месяцев: ...
```

## Правила

- Не записывать ров как «сильный», если нет конкретного факта/метрики. Лозунги типа «отличный UX» — это не ров.
- Минимум 2 ключевых рва — один ров редко держит продукт.
- Если на всех 7 типах «слабо / нет» — это честный сигнал: продукт копируется, фокус должен быть на build moat, а не на scale.
