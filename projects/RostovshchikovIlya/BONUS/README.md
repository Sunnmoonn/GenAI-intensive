# BONUS — Скил «Анализ продукта по слоям»

Пошаговый скил, который берёт любой продукт (чужой или свой) и раскладывает его на слои — от внешнего UX до внутреннего moat. Нужен, чтобы быстро понимать конкурентов, оценивать клоны и честно смотреть на собственный продукт.

## Зачем

Когда фаундер или команда анализирует продукт, чаще всего получается либо «нравится / не нравится» (UX-обзор), либо «бизнес-модель крутая» (инвесторский pitch). Оба взгляда неполные: первый не видит рва, второй не видит, почему пользователь реально приходит и остаётся.

Этот скил принуждает разобрать продукт на 5 слоёв и ответить на конкретные вопросы по каждому. Результат — одностраничник, который одинаково читается и маркетологом, и техническим фаундером, и инвестором.

## Слои

| # | Слой | Главный вопрос |
|---|------|----------------|
| 1 | UX / интерфейс | Что видит и чувствует пользователь в первые 30 секунд? |
| 2 | Ценность / JTBD | На какую работу пользователь нанимает продукт? |
| 3 | Технология / архитектура | Какая механика делает эту работу возможной и масштабируемой? |
| 4 | Бизнес-модель | Как продукт превращает ценность в выручку и каковы unit-экономика/каналы? |
| 5 | Рвы (moats) | Что удержит пользователя и отпугнёт конкурента через 12 месяцев? |

## Скилы пайплайна

| Скил | Когда | Что делает |
|------|-------|-----------|
| [PA_Surface](pa-surface.md) | Вход — URL, скриншоты, demo | Снимает UX-слой и JTBD: первые впечатления, главные сценарии, сегменты |
| [PA_Mechanism](pa-mechanism.md) | После surface | Разбор технологии и бизнес-модели: как работает внутри, на чём зарабатывает |
| [PA_Moat](pa-moat.md) | После mechanism | Поиск и приоритизация рвов: данные / привычка / сеть / железо / бренд / дистрибуция |

## Пайплайн

```
Продукт (URL / demo / питч)
    → PA_Surface   → карточка UX + JTBD
    → PA_Mechanism → карточка технологии + бизнес-модели
    → PA_Moat      → карточка рвов с приоритизацией
    = Готовый one-page анализ продукта по 5 слоям
```

Каждый скил работает и автономно: если надо быстро оценить только moat у конкурента — можно запустить только `PA_Moat`, передав ему короткое описание.

## Как использую лично

- Перед копированием чужого продукта (как PROJECTCLONE doomlingo.ai) — прогоняю все 3 скила, чтобы понять, что именно копировать, а что — нет.
- Раз в квартал — прогоняю по собственному Easy Parking, смотрю, где рвы слабые.
- При появлении нового конкурента (Tele2 СмартПаркинг, Паркоматика и т.п.) — разбираю по слоям за 30 минут, кладу карточку в vault.
