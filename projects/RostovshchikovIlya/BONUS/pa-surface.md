# PA_Surface — анализ UX + JTBD

## Когда запускать

Первый шаг анализа любого продукта. Вход — URL, скриншоты, короткое demo-видео или live-демо.

## Что делает

Снимает два верхних слоя продукта: UX (что видит пользователь) и JTBD (на какую работу его нанимают). Результат — структурированная карточка, с которой можно идти на следующий шаг.

## Шаги

1. **First impression (30 секунд).** Описать: что бросается в глаза, какое обещание транслирует главный экран, есть ли понятный CTA. Записать одно ощущение словами пользователя («просто», «сложно», «дорого», «играбельно»).
2. **Core loop.** Найти главный пользовательский цикл: вход → действие → результат → возврат. Описать в 3–5 шагах.
3. **Сегменты.** Для кого этот loop работает лучше всего? Указать 2–3 сегмента с критериями (возраст, контекст, триггер).
4. **JTBD.** Сформулировать 1–3 джоба по шаблону «Когда …, я хочу …, чтобы …». Для каждого — контекст, мотивация, результат.
5. **Конкуренты на эту работу.** Перечислить, чем ещё пользователь закрывает эту же работу (включая «ничего не делает»).
6. **Узкие места UX.** 2–3 места, где пользователь скорее всего отвалится.

## Выход

Markdown-карточка:

```
# <Продукт> — Surface

## First impression
...

## Core loop
1. ...
2. ...

## Сегменты
- ...

## JTBD
### Джоб 1
- Когда: ...
- Хочу: ...
- Чтобы: ...

## Конкуренты на работу
- ...

## UX-риски
- ...
```

## Правила

- Не додумывать — если не видно из demo/URL, пометить `?` и идти дальше.
- Сегменты формулировать через поведение, а не через демографию («водители в платной зоне ежедневно», а не «мужчины 30+»).
- JTBD писать от первого лица пользователя, без маркетингового словаря.
- Максимум 1 страница — иначе не читается.
